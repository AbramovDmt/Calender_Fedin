## 1. Что это и для кого

**«Мой Календарь»** — дневной календарь-ежедневник для хозяина, который сдаёт один загородный домик и параллельно ведёт мелкие сторонние дела.

- **Один пользователь**, своё устройство.
- **Календарь по дням**, не по часам.
- В одной клетке дня видно сразу: **занятость домика, деньги, дела, даты, напоминания.**
- Главная ценность: **занятость и доход — прямо на сетке месяца.**

### Принципы (не нарушать)
1. **Проще бумаги.** Любое действие — минимум тапов. Если фича добавляет трения — её нет.
2. **Один домик.** Никакой мульти-объектности, синхронизаций каналов, аккаунтов.
3. **Офлайн и локально.** Данные на устройстве, интернет не нужен.
4. **Скорость важнее количества фич.** Мгновенный отклик на телефоне.
5. **Данные принадлежат хозяину.** Свободный экспорт/импорт.

## 2. Платформа и технологии

| | Решение |
|---|---|
| Тип | Веб-приложение, **PWA** (устанавливается на телефон, работает офлайн) |
| Приоритет | **Mobile-first** (телефон). Десктоп — просто корректно адаптируется. |
| Стек | **Vanilla JS + HTML + CSS**, без фреймворков и сборщика |
| Зависимости | Минимум. Даты — строки `YYYY-MM-DD` (не объект `Date`, чтобы не ловить сдвиги часовых поясов). Иконки — Lucide (инлайн SVG). |
| Хранилище | **IndexedDB** (надёжнее `localStorage`, нет лимита ~5 МБ, не вычищается ITP у установленной PWA) + запрос `navigator.storage.persist()` |
| Перенос данных | Экспорт/импорт `.json`; опц. человекочитаемый `.csv`; напоминание делать бэкап |
| Язык | Русский |
| Валюта | Рубль (₽) |

### PWA-требования
- `manifest.json` (имя, иконки, `display: standalone`, `theme_color`/`background_color` под тему).
- Service worker: precache статики, офлайн-работа, стратегия cache-first.
- `viewport-fit=cover` + `env(safe-area-inset-*)` для шапки и нижней зоны.
- Перехват `beforeinstallprompt` → собственная кнопка «Установить». **Установка активно поощряется** — у установленной PWA данные не вычищаются (см. §9 Риски).
- При первом сохранении запрашиваем `navigator.storage.persist()` (стойкое хранилище).
- В standalone у каждого экрана/sheet своя кнопка «назад/закрыть» (системной нет).
- `overscroll-behavior-y: contain` (убрать pull-to-refresh).

## 3. Модель данных

Всё приложение — один документ в **IndexedDB** (хранилище `myCalendar`, версия схемы в `version`). Автосохранение: каждое изменение сразу пишется в базу.

```jsonc
{
  "version": 1,
  "settings": {
    "theme": "system",          // "light" | "dark" | "system"
    "currency": "₽",
    "notifications": false,     // ползунок уведомлений напоминаний (best-effort)
    "lastBackupAt": "ISO?"      // для напоминания «пора сделать бэкап»
  },
  "records": [ /* массив записей, см. ниже */ ],
  "clients": [                  // список клиентов для автоподстановки
    { "name": "string", "phone": "string?", "lastUsedAt": "ISO" }
  ]
}
```

### Общие поля записи
```jsonc
{
  "id": "uuid",
  "type": "booking | bath | work | event | reminder",
  "createdAt": "ISO",
  "updatedAt": "ISO",
  "comment": "string?"
}
```

### 🏠 booking — Бронь домика (диапазон дней)
```jsonc
{
  "type": "booking",
  "guestName": "string",
  "phone": "string?",
  "checkIn": "YYYY-MM-DD",     // день заезда (встретить)
  "checkInTime": "14:00",     // время заезда — ключевая точка приёма
  "checkOut": "YYYY-MM-DD",    // день выезда (проводить), ночь не учитывается
  "checkOutTime": "12:00",    // время выезда — приём/проверка домика
  "price": 18000,             // стоимость аренды, ₽ (хозяин ставит сам)
  "extras": {                 // что входит — галочки, без денег
    "sauna": false,           // 🧖 баня
    "bikes": false,           // 🚲 велосипеды
    "sup": false              // 🛶 сапы
  },
  "deposit": { "taken": false, "amount": 0 },  // переключатель + сумма (не доход)
  "comment": "string?"
}
```
- **Деньги:** `price` засчитывается в доход на дату `checkIn`.
- **Занятость:** ночи с `checkIn` (вкл.) по `checkOut` (искл.).
- `extras` — только визуальные отметки «что входит в заезд», денег за ними нет.
- `checkInTime` / `checkOutTime` — важны операционно (когда встречать / закрывать), желательны, но не блокируют сохранение.

### 🧖 bath — Баня (без домика, один день)
Самостоятельное операционное событие: гостей надо встретить и после проверить порядок. Баня почасовая (минимум ~3 часа), поэтому **время обязательно** — хозяину ехать ~30 мин, нужно знать, когда встречать и закрывать.
```jsonc
{
  "type": "bath",
  "guestName": "string?",
  "phone": "string?",
  "date": "YYYY-MM-DD",
  "fromTime": "16:00",        // обязательно — начало
  "toTime": "19:00",          // обязательно — конец
  "price": 5000,              // ₽, доход (хозяин ставит сам, без расчёта по часам)
  "extras": {                 // при бане есть кухня/зона отдыха — могут и покататься
    "bikes": false,           // 🚲 велосипеды
    "sup": false              // 🛶 сапы
  },
  "deposit": { "taken": false, "amount": 0 },  // как в броне
  "comment": "string?"
}
```
- **Деньги:** `price` засчитывается в доход на дату `date`.
- **Занятость:** баня — не ночь; в «занятых ночах» не учитывается (это дневное событие).

### 💼 work — Работа / сторонний заработок
Полноценное дело: может быть привязано к человеку, поездке и времени.
```jsonc
{
  "type": "work",
  "title": "string",
  "date": "YYYY-MM-DD",
  "time": "14:00",            // когда (поездка/приём), необязательно
  "contactName": "string?",   // с кем взаимодействует
  "phone": "string?",
  "amount": 7000,             // ₽, доход
  "comment": "string?"
}
```

### 📅 event — Событие (без денег)
```jsonc
{
  "type": "event",
  "title": "string",          // "День рождения Андрея", "Встреча"
  "date": "YYYY-MM-DD",
  "time": "12:00",            // необязательно; пусто = весь день (напр. день рождения)
  "contactName": "string?",   // к кому привязано
  "phone": "string?",
  "comment": "string?"
}
```

### 🔔 reminder — Напоминание (без денег)
```jsonc
{
  "type": "reminder",
  "title": "string",          // "Купить уголь"
  "date": "YYYY-MM-DD",
  "time": "09:00",            // когда напомнить, необязательно
  "notify": false,            // ползунок: пробовать прислать уведомление (best-effort)
  "done": false,              // галочка выполнения
  "comment": "string?"
}
```

### Правила расчётов
- **Доход за день** = (`booking.price` где `checkIn` = день) + (`bath.price` где `date` = день) + (`work.amount` где `date` = день).
- **Доход за неделю** = сумма доходов дней недели (показывается в конце строки месяца).
- **Доход за месяц** = сумма доходов всех дней месяца.
- Доход брони привязан к дате заезда целиком, даже если бронь переходит в другой месяц (осознанное упрощение).
- **Депозит в доход не входит** (отдельная отметка «взят / нет» + сумма).
- **Занятых ночей в месяце** = число ночей броней домика, попавших в месяц (ночь принадлежит дате своего начала; баня не считается ночью).
- **Занятость, %** = занятых ночей ÷ число дней в месяце × 100.
- **Пересечение броней:** при сохранении брони, чьи ночи пересекаются с существующей, показываем **мягкое предупреждение** (не запрет): «На эти дни уже есть бронь {имя}».

## 4. Дизайн-система «Тёплый дом»

Тёплая «бумажная» основа + Material 3 Expressive-формы + тонкое стекло на плавающих элементах. Не холодный серый, не чисто чёрный/белый.

### 4.1 Цвета (CSS-переменные)

**Светлая тема**
```css
--bg:            #FAF6F0;  /* фон приложения, тёплый off-white */
--surface:       #FFFCF8;  /* карточка */
--surface-2:     #F3ECE2;  /* поднятая поверхность / hover */
--border:        #E7DDD0;
--text:          #2B2521;  /* основной (тёплый near-black) */
--text-muted:    #7A6F64;  /* вторичный */
--accent:        #D9603F;  /* терракота — бронь домика */
--sauna:         #B5772C;  /* медово-янтарный — баня (отдельный тип) */
--success:       #5E7D5A;  /* деньги / занято (мох) */
--warning:       #E3A23B;  /* ожидание / депозит */
--danger:        #C4553D;  /* выезд / удаление */
```

**Тёмная тема**
```css
--bg:            #1A1614;  /* тёплый уголь */
--surface:       #221D1A;  /* элевация 1 */
--surface-2:     #2B2421;  /* элевация 2 */
--surface-3:     #332A26;  /* элевация 3 */
--border:        #3A312C;
--text:          #EFE7DE;  /* тёплый белый (не #FFF) */
--text-muted:    #A99C8F;
--accent:        #E8795C;  /* терракота десатурированная — бронь */
--sauna:         #D9A24A;  /* медово-янтарный (ярче) — баня */
--success:       #86B97E;  /* деньги ярче, чтобы «попадали» */
--warning:       #F4C542;
--danger:        #D9603F;
```

- Элевация в тёмной — **через светлый тинт поверхности**, не тенями.
- `<meta name="theme-color">` и CSS `color-scheme` обновляются под тему.

### 4.2 Типографика
- **Крупные заголовки и большая сумма месяца:** `Unbounded` (характерный, дисплейный). Только крупно — для мелкого он плохо читается. Запасной — `Geologica`.
- **Текст, UI, все числа в сетке и формах:** `Manrope` (кириллица, премиальный на мелком, есть tabular-nums). Запасной — `Golos Text`.
- Деньги и номера дней — **табличные цифры** (`font-variant-numeric: tabular-nums`); крупные суммы — тяжёлый вес.
- Шкала: H1 28 / H2 22 / H3 18 / body 16 / small 14 / caption 12.

### 4.3 Размеры, скругления, отступы
- Радиусы: карточки **20px**, sheets/hero **24–28px**, кнопки/чипы — **pill**, мелкие чипы **12px**.
- Сетка отступов: шаг **4px** (4/8/12/16/24/32).
- **Тап-цель ≥ 44–48px** (даже если видимый маркер мельче).
- Контраст выше минимума WCAG AA (на солнце). **Никогда только цветом** — цвет + форма/иконка/подпись.

### 4.4 Иконки и эмодзи
- UI-действия — **Lucide** (outline по умолчанию, filled в активном состоянии).
- Эмодзи типов (🏠 бронь, 🧖 баня, 💼 работа, 📅 событие, 🔔 напоминание) — **только в детали дня и формах**. Галочки «что входит»: 🧖 баня (внутри брони), 🚲 велики, 🛶 сапы.
- **В сетке месяца эмодзи НЕ используем** (разный рендеринг, шум на мелком) — только цветные маркеры / SVG-иконки Lucide. Различаем элементы формой, а не только цветом: бронь — **полоса** `--accent`, баня — **одиночный чип** `--sauna` с мини-иконкой, прочее — точки.

### 4.5 Стекло и движение
- Тонкое стекло (`backdrop-filter: blur(20px)` + тёплый тинт) — только шапка, bottom sheet, FAB. **Под стеклом не должно быть важного текста**; тинт достаточно плотный, чтобы текст НА стекле читался на солнце (контраст важнее эффекта). Где стекло мешает читаемости — делаем непрозрачную поверхность.
- Переходы 200–500 мс, пружинные; press-состояние клеток; shape-morph при выборе дня.
- Свайп месяцев с инерцией. **Всегда уважать `prefers-reduced-motion`.**

## 5. Экраны

### 5.1 Главный экран — Месяц
**Назначение:** одним взглядом — занятость, деньги, дела за месяц.
**Настроение:** спокойный «бумажный» разворот ежедневника; тепло, чисто, читаемо.

**Принцип плотности:** клетка не перегружается. Видно главное — **есть ли бронь** и **есть ли внутри дела**. Подробности и точные суммы — по тапу в деталь дня. Деньги читаем агрегатами (неделя/месяц), а не выискиваем по клеткам.

**Обязательные элементы:**
1. **Шапка (стеклянная, sticky):** название месяца + год (`Unbounded`), стрелки ‹ ›, кнопка **«Сегодня»**, иконка настроек.
2. **Строка дней недели** ПН…ВС (ПН — первый).
3. **Сетка по неделям (строки).** В клетке дня — приоритет сверху вниз:
   - номер дня (у сегодня — заполненное кольцо акцентом); дни соседних месяцев приглушены;
   - **полоса брони** заезд→выезд (скруглённые торцы; пересменок — диагональ), цвет `--accent` — главный элемент;
   - **баня** — одиночный чип `--sauna` с мини-иконкой;
   - **есть прочие дела** (работа/событие/напоминание) — компактные точки по цвету типа; если не влезают — одна точка-«есть дела» / «+N». Не пытаемся показать всё.
   - **сумма дохода** — мелко и только в «денежные» дни с заметной суммой; на остальных не показываем (порог сокращения, напр. «18к», задаётся в реализации).
4. **Итог недели:** в конце каждой строки (или мелкой подписью под ней) — **доход за неделю** `--success`.
5. **Итоговая плашка месяца (bento, внизу/закреплена):** «Доход за месяц» (крупно, `Unbounded`, `--success`, табличные цифры), «Занято ночей / Занятость %», «Депозиты» (если есть).
6. **FAB** снизу-справа — «+ Добавить». Создаёт запись с **сегодняшней датой** по умолчанию (дату можно изменить в форме).

**Поведение:** свайп влево/вправо — смена месяца; тап по дню — открыть деталь дня (sheet); тап по FAB — выбор типа записи (дата = сегодня). Основной путь добавления на конкретную дату — тап по дню → «+».

### 5.2 Деталь дня — Bottom Sheet
**Назначение:** посмотреть и изменить всё за конкретный день.
**Настроение:** быстрый, лёгкий, выезжает снизу в зону большого пальца.

**Обязательные элементы:**
- «ручка» перетаскивания (peek → раскрыт), кнопка закрыть;
- дата заголовком (день недели, число, месяц);
- **список записей дня** карточками: эмодзи типа + название + время (если задано) + сумма (если есть) + контакт (если задан). Для брони — отметка «заезд (14:00)» / «выезд (12:00)» / «проживание», горящие галочки «что входит» (🧖🚲🛶), депозит. Для бани — время «с–по», галочки 🚲🛶, депозит. У напоминаний — чекбокс.
- записи дня сортируются по времени (без времени / «весь день» — сверху).
- мини-итог дня (доход за день);
- кнопка **«+ Добавить запись»** → выбор типа (дата = выбранный день).
- свайп по карточке или долгий тап → редактировать / удалить. **Удаление — с подтверждением** (диалог или undo-тост «Удалено · Отменить»).

### 5.3 Формы создания/редактирования
Общий стиль: полноэкранный лист или раскрытый sheet, крупные поля под палец, кнопки **«Сохранить»** (акцент, внизу, в safe-area) и «Отмена/Удалить». Дата/время — **нативные** `input[type=date|time]`.

**Автоподстановка клиентов:** в поле имени гостя/контакта — подсказки из списка `clients` (повторные гости: начал набирать «Лиг…» → выбрал, телефон подставился). При сохранении новый контакт добавляется/обновляется в `clients`.

- **🏠 Бронь:** имя гостя*, телефон, заезд* + время заезда, выезд* (≥ заезд+1) + время выезда, стоимость*, комментарий.
  - **«Что входит»** — три галочки-тумблера: 🧖 баня · 🚲 велики · 🛶 сапы (без денег, просто отметка).
  - **Депозит** — тумблер «взят» + поле суммы (появляется при включении). В доход не идёт.
  - Подсказка: «N ночей».
- **🧖 Баня:** имя гостя, телефон, дата*, **время с* — по***, стоимость*, комментарий.
  - **«Что входит»** — галочки 🚲 велики · 🛶 сапы.
  - **Депозит** — как в броне.
- **💼 Работа:** название*, дата*, время, контакт (имя), телефон, сумма*, комментарий. (Сюда же — разовые продажи вроде «баня соседу», велик напрокат и т.п.)
- **📅 Событие:** название*, дата*, время (пусто = «весь день», напр. день рождения), контакт (имя), телефон, комментарий.
- **🔔 Напоминание:** текст*, дата*, время, **тумблер «уведомить»** (best-effort), комментарий; чекбокс «выполнено».

(\* — обязательные поля; кнопка «Сохранить» неактивна, пока не заполнены. Время заезда/выезда у брони желательно, у бани — обязательно. Время у работы/события/напоминания необязательно: пусто = весь день / без времени.)

### 5.4 Настройки
**Обязательные элементы:**
- **Тема:** сегмент Светлая / Тёмная / Системная.
- **Уведомления напоминаний:** ползунок вкл/выкл (запрашивает разрешение; работает best-effort, см. §9).
- **Данные:** «Выгрузить в файл» (`.json`), «Загрузить из файла» (с подтверждением замены), опц. «Экспорт в CSV». Показываем дату последнего бэкапа и **ненавязчивое напоминание сделать бэкап**, если давно не делали.
- **Установить на телефон** (если доступно) — с пояснением, что так данные надёжнее сохраняются.
- О приложении / версия.

### 5.5 Пустые состояния и онбординг
- Первый запуск: короткий приветственный экран (1 экран) — «Это ваш ежедневник по дням» + кнопка «Начать».
- Пустой день в sheet: дружелюбная подсказка + кнопка «Добавить».
- Пустой месяц: сетка без записей — это нормально, ничего не навязываем.

## 6. Нефункциональные требования
- **Скорость:** открытие < 1 c (после установки PWA), отклик на тап мгновенный.
- **Надёжность данных:** автосохранение в IndexedDB при каждом изменении; `storage.persist()`; при импорте — валидация и резервная копия предыдущего состояния перед заменой. Подробнее — §9 «Риски и решения».
- **Доступность:** тап-цели 44–48px, контраст AA+, не только цвет, `prefers-reduced-motion`, читаемость на солнце.
- **Адаптив:** корректно от 320px шириной; на десктопе — центрированная колонка.

## 7. Объём MVP и этапы

**Этап 1 — ядро (MVP):**
1. Хранилище на IndexedDB + автосохранение + `storage.persist()`.
2. Сетка месяца по неделям + навигация (свайп, «Сегодня»), тема Светлая/Тёмная/Системная.
3. Бронь: создание/редактирование/удаление (с подтверждением), полоса заезд→выезд, время, галочки «что входит», депозит, цена; предупреждение о пересечении.
4. Итог недели + итог месяца (доход, ночи, занятость %).
5. Деталь дня (bottom sheet), список и действия; FAB (дата = сегодня).

**Этап 2 — полный набор записей:**
6. Баня (время с–по, галочки, депозит), Работа, Событие, Напоминание (чекбокс + тумблер «уведомить»).
7. Маркеры/точки, чип бани и «+N» в клетке; список клиентов + автоподстановка.

**Этап 3 — перенос и упаковка:**
8. Экспорт/импорт JSON, опц. CSV; напоминание о бэкапе.
9. PWA: manifest, service worker, установка, офлайн, safe-area; уведомления напоминаний (best-effort).
10. Онбординг, пустые состояния, полировка анимаций.

## 8. Чего НЕ делаем (явно вне скоупа)
Синхронизация с Booking/Airbnb/Суточно · мульти-объект (второй домик) · динамические/сезонные цены, RevPAR-аналитика · авто-сообщения гостям · умные замки/удалённое заселение · уборочный персонал, роли · аккаунты, облако, подписки · недельный/часовой/годовой виды (пока только месяц) · статус «предварительная бронь» (хозяин подтверждает из официальных источников) · учёт возврата депозита (пока только отметка «взят»).

## 9. Риски и решения

| Риск | Решение |
|------|---------|
| **Потеря данных** — браузер вычищает хранилище (на iOS Safari — ~7 дней без открытия по ITP). Для бизнес-данных критично. | IndexedDB + `navigator.storage.persist()`; активно поощряем **установку PWA** (у установленной лимита нет); **напоминание делать бэкап** (экспорт JSON) + дата последнего бэкапа; всё локально, без облака. |
| **Двойная бронь** на один домик. | Мягкое предупреждение при пересечении дат (не запрет): «На эти дни уже есть бронь {имя}». |
| **Напоминания не сработают** как push — веб-уведомления на iOS ненадёжны. | Напоминания всегда **видны в приложении** (в дне и в списке «на сегодня»); push — **best-effort** по тумблеру, без гарантии. Ожидание управляется честно в UI. |
| **Перегруженная клетка** дня. | Жёсткий приоритет содержимого клетки, агрегаты денег по неделе/месяцу, детали — по тапу (§5.1). |
| **Сдвиги дат из-за часовых поясов.** | Даты — строки `YYYY-MM-DD`, без объекта `Date` для хранения. |
| **Случайное удаление.** | Подтверждение / undo-тост. |
| **Нечитаемо на солнце** (стекло, тонкий серый, «только цвет»). | Контраст выше AA, стекло без текста под ним, различие формой + цветом (§4.4–4.5). |
