# Research — Мой Календарь

Исследование перед разработкой: похожие приложения, UX-тренды, визуальный дизайн (2025–2026).
Цель — сделать удобно, красиво, не банально, и не наступить на чужие грабли.

---

## Часть 1. Похожие приложения и фишки

### Что уже есть на рынке

**Российские (посуточная аренда) — ближайший рынок:**
- **RealtyCalendar**, **RentSkill**, **MaxiBooking**, **Bnovo / TravelLine** — облачные «шахматки» с синхронизацией каналов (Booking/Airbnb/Суточно), для тех, у кого много объектов. Нам это **оверкилл**.
- **Планировщик Аренды** (iOS) — самый близкий аналог для одного хозяина: шахматка, доходы/расходы, напоминания об оплате, депозиты, экспорт в Excel, цветовое кодирование объектов, тёмная тема.

**Зарубежные lite-инструменты:**
- **BedBooking** — работает **онлайн и офлайн**, единый календарь, **доп.услуги прикрепляются к броне** (доп.кровать, еда, налог), статистика занятости. По фичам ближе всего к нам.

**«Деньги на сетке» (наша фишка):**
- **Money Calendar** — суммы дохода/расхода **прямо в клетках дня** + бегущий итог сверху. Это наш ближайший аналог по главной ценности.
- **CalendarBudget, PocketSmith** — деньги и прогноз на календаре.

### Killer-feature (что реально ценят)
- **Занятость + деньги одним взглядом, без лишних тапов.** Большинство rental-приложений прячут деньги, а money-приложения не показывают занятость. **Объединение обоих на одной сетке месяца — наш дифференциатор** (и он подтверждён рынком).
- **«Замена бумажного блокнота»** — главный эмоциональный крючок. Побеждает тот, у кого *меньше* трения, чем у бумаги, а не больше функций.

### Чужие грабли (чего избегать)
1. **Синхронизация с Booking/Airbnb** — источник жалоб №1 («двойные брони, нет поддержки»). Нам не нужна (мы офлайн, один домик).
2. **Рост цен / навязанные подписки** — пользователи хотят простой инструмент без аккаунта и абонплаты.
3. **Раздутость** — полноценные PMS «ощущаются как оверкилл» для соло-хозяина.
4. **Тормоза** — на телефоне ощущение скорости важнее количества фич.
5. **Авто-смена статусов**, которые хозяин ждёт контролировать руками.
6. **Лок-ин данных** — люди хотят выгрузку (валидирует наш JSON + желательно человекочитаемый экспорт).

### Что взять (в рамках нашего скоупа)
1. Деньги + занятость на одной сетке месяца + **месячный итог** (доход, ночей занято / свободно).
2. **Бронь как полоса** заезд→выезд с ценой.
3. **Услуги как доп.позиции** — к броне или отдельно.
4. **Эмодзи/цвет по типам** для мгновенного сканирования.
5. **Датированные напоминания** (без планировщика-движка).
6. **Экспорт JSON + человекочитаемый** (Excel/CSV/сводка) — доверие, анти-лок-ин.

### Жёстко НЕ делаем
Синхронизацию каналов · мульти-объект («а вдруг второй домик») · движки динамических цен/RevPAR · авто-сообщения гостям · умные замки · уборочный персонал · аккаунты/облако/подписки.

## Часть 2. UX и удобство (mobile-first, 2025–2026)

### Сетка месяца + плотная информация в клетке
- **Многодневная бронь = непрерывная горизонтальная полоса** (не точки!). Точки рвут связь «эти дни — одна бронь». Полоса = диапазон; **точки — для одиночных событий** (услуга/работа/событие/напоминание).
- **Заезд/выезд = скруглённые «торцы» полосы**, а не только цвет. День-пересменок (выезд утром + заезд вечером) — **диагональное деление клетки** (паттерн отельных шахматок). Это прямо про «особые дни заезда/выезда».
- **Деньги — мелко и только когда есть.** Маленькое число внизу клетки, на пустых днях не показывать. Большой итог — в шапке/подвале месяца.
- **Цвет = тип, форма = вид.** Пять эмодзи в клетке 50px — много; лучше маленькие цветные маркеры в сетке, а сами эмодзи — в детали дня. При переполнении — «+N».

### Эргономика одной руки
- **Низ экрана — рабочая зона**, верх — «трудная». Сетка (читаем) — сверху/центр, действия — снизу.
- **Деталь дня = bottom sheet** (выезжает снизу, не теряет контекст), перетаскиваемый (peek → раскрыт). Не модалка по центру и не отдельная страница.
- **Добавить запись = FAB** снизу (по центру — удобно обеим рукам).
- **Навигация по месяцам = свайп** влево/вправо + видимые стрелки + кнопка «Сегодня».

### Микро-взаимодействия (тренд, но со вкусом)
- **Брать:** пружинный bottom sheet, лёгкое press-состояние клеток, плавный свайп месяцев, контекстный хаптик (сохранение, «прилипание» при растягивании брони). 200–500 мс, физика.
- **Избегать:** декоративные анимации «просто так», длиннее 500 мс, блокирующие действие. Всегда уважать `prefers-reduced-motion`.

### Светлая/тёмная тема (UX)
- Каскад: **localStorage (выбор) → системная (`prefers-color-scheme`) → дефолт**.
- Лучше три состояния: **Светлая / Тёмная / Системная**.
- Применять через `data-theme` + CSS-переменные; тему ставить **инлайн-скриптом в `<head>` до первой отрисовки** (без мигания). Обновлять `<meta name="theme-color">` и CSS `color-scheme`.
- Никогда не чисто чёрный/белый.

### Доступность и читаемость на улице
- Тап-цель **44–48px** (даже если маркер мельче).
- Контраст **выше** минимума (на солнце AA в помещении может не хватать); тонкий серый текст на сером — провал.
- **Никогда только цветом** (заезд/выезд/тип) — цвет + форма/иконка.

### PWA
- `viewport-fit=cover` + `env(safe-area-inset-*)` — чтобы низ (FAB) не прятался под home-indicator, а шапка — под чёлкой.
- Свой install-кнопкой (перехват `beforeinstallprompt`), не сырой баннер.
- В standalone **нет кнопки «назад» браузера** → у каждого экрана/sheet своя «закрыть/назад».
- Офлайн-ощущение: скелетоны вместо спиннеров, `overscroll-behavior-y: contain` (убрать pull-to-refresh).

## Часть 3. Визуальный дизайн и темы (2025–2026)

### Тренды
- **IN:** зрелый glassmorphism (тонкое стекло только на «хроме» — шапка/sheet/FAB), **Material 3 Expressive** (смелый акцент, вариативная типографика, крупные скругления, пружинная анимация), **bento-сетки** для сводки, большие радиусы (16–28px), выразительные заголовки + спокойный текст, лёгкое зерно/тёплые градиенты («рукотворность», уют).
- **Dated/банально (избегать):** холодный корпоративный серый (клон Notion/Asana), стекло везде, плоский Material 2 с жёстким синим, дефолтный Bootstrap/Tailwind «синее-на-белом» с мелкими радиусами.

### Цвет — «уютно, но современно»
Доминанта 2026 — тёплая, землистая палитра. Идеально под «хозяин загородного домика».

**Акцент: терракота** `#D9603F` (светлая) / `#E8795C` (тёмная).
**Деньги/занятость:** приглушённый мох-зелёный `#5E7D5A` / `#86B97E` (не неон).

**Светлая тема:**

### Типографика (Google Fonts, с кириллицей и ₽)
- **Вариант 1 (мой выбор):** заголовки/числа — **Unbounded** или **Geologica**; текст/UI — **Manrope** (автор — кириллический дизайнер).
- **Вариант 2 (самый «родной русский», максимально читаемый):** **Golos Text** на всё.
- Для денег — тяжёлый вес + **табличные цифры** (выравнивание).

### Тёмная тема «как надо»
Не `#000` (смазывание на OLED) · элевация светлым тинтом поверхности, не тенями · не чисто белый текст · акцент в тёмной чуть **десатурировать** · деньги — ярче зелёным + тяжёлый вес.

### Иконки
- **Lucide** (минимал, stroke) — outline по умолчанию, **filled в активном** состоянии. Либо **Phosphor** (есть duotone). Системные действия — иконки (не эмодзи); типы записей — маленький курируемый набор эмодзи/duotone для тёплого личного ощущения.

### Референсы (что подсмотреть)
**Amie** (живые, но осмысленные анимации, цветные чипы) · **Cron / Notion Calendar** (чистейшая плотная сетка, скорость ввода) · **Timepage** (типографика дат, один сигнатурный акцент) · **Floret / Structured** (уютный спокойный планер) · **CalendarBudget** (деньги-на-календаре).

## Итоговое дизайн-направление: «Тёплый дом» (Warm Hearth)

Тёплый Material 3 Expressive + тонкое стекло только на плавающих элементах.
Отличается от шаблонов тем, что вместо холодного серого — тёплая «бумажная» основа, но при этом чётко и современно.

- **Акцент:** терракота `#D9603F` / `#E8795C`.
- **Фон:** `#FAF6F0` / `#1A1614` (никогда чисто белый/чёрный), опц. лёгкое зерно.
- **Деньги/занятость:** мох-зелёный, тяжёлый вес, табличные цифры, на отдельной плитке.
- **Шрифты:** Unbounded (или Geologica) для заголовков/чисел + Manrope для текста.
- **Радиусы:** 20px карточки, 24–28px sheets, pill-кнопки/чипы, 12px мелкие чипы.
- **Движение:** крупные скругления, пружинные переходы, shape-morph при выборе дня.
- **Стекло:** тонкое (`backdrop-filter: blur(20px)` + тёплый тинт) на шапке, нижнем sheet, FAB.
- **Иконки:** Lucide outline → filled в активном; duotone-акцент для типов записей.
- **Свет vs тьма:** свет = тёплая бумага + мягкие тени; тьма = тёплый уголь + лестница тинтов поверхности, десатурированный акцент, яркий зелёный для денег, тёплый белый текст. Одна тёплая гамма в обеих темах.

## Что это меняет в нашей концепции
- **Бронь рисуем полосой** (не значком 🏠) с торцами заезд/выезд и диагональю на пересменке. ➜ обновить раздел 4 в CONCEPT.md.
- **Эмодзи типов — в детали дня**, в сетке — цветные маркеры/точки + «+N».
- **Деталь дня — bottom sheet**, добавление — FAB снизу.
- Добавить **тему 3-состояния** и пресет палитры «Тёплый дом».
- Рассмотреть **человекочитаемый экспорт** (CSV/сводка) в дополнение к JSON.
